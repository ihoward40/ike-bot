# Contributing Guidelines for Neo-Commonwealth Legal Strategy Automation System

Thank you for considering contributing to the Neo-Commonwealth Legal Strategy Automation System! To ensure a smooth collaboration, please follow these contributing guidelines:

## Code Style Standards
- Follow the [Google Java Style Guide](https://google.github.io/styleguide/jsguide.html) for JavaScript.
- Maintain consistent indentation (2 spaces) and line length (maximum of 80 characters).
- Use meaningful variable and function names.

## PR Review Checklist
- Ensure the PR description clearly explains the purpose and changes made.
- Reference the relevant issue number (if applicable).
- All tests must pass before the PR is submitted for review.
- Check for any code style issues using the linter.

## Testing Requirements
- Write unit tests for any new features or bug fixes.
- Ensure that all existing tests pass after changes.
